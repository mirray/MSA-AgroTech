### <a name="_qmphm5d6rvi3"></a>**Решение**
Для автоматизации свинокомплекса будет построена отдельная система - АвтоТехноСвиноКомплекс (далее АТСК) 

На ферме может быть не стабильный Wi-Fi (+R6), при этом необходимо оперативное реагирование на события, происходящие на ферме (P3, R1)

На каждой ферме могут быть расположены Edge - устройства, но сервер должен быть только один на весь комплекс (+R1, +R8)

Исходя из этого система может быть разделена на три слоя:  

- Edge-системы - edge-устройства расположенные непосредственно на ферме (не должны быть зависимы от интернета, важна высокая производительность)
- Синхронизация данных с сервером
- Сервисы, непосредственно на общем сервере - структурирование данных с ферм и их аналитика

Система может быть разделена на несколько контейнеров:
 - Edge-системы:
    - Видеонаблюдение - Анализ видео с камер наблюдения с помощью нейронной сети
    - Умные устройства - Управление и анализ данных с умных устройств
 - Синхронизация данных с сервером
    - Шлюз Событий - Ведет учет событий происходящих на ферме
 - Сервисы на сервере:
    - Учет животных - Учет состояния животных
    - Внутренняя аналитика - Вычисление метрик и прогнозирование

При таком разделении с edge-системами взаимодействуют люди, находящиеся непосредственно на ферме (Операторы, Техники). Сервисы на сервере доступны для пользователей, которые могут находится вне фермы.

### <a name="_bjrr7veeh80c"></a>**Альтернативы**

Критичным местом такой архитектуры будет Шлюз событий. Прри его сбое или отказе, сервер не будет получать никаких данных с фермы.

Чтобы повысить надежность и масштабируемость системы возможно отказаться от шлюза событий и предоставить каждой Edge - системе возможность самостоятельно синхронизировать свои данные с сервисами на сервере.

**Недостатки, ограничения, риски**

Недостатки альтернативного решения:
 - Усложнение архитектуры - синхронизация данных напрямую между сервисами требует детального документирования и согласования
 - Сложность расширения - при внедрении нового функционала нужно будет перестраивать потоки сообщений между различными частями системы, лиюо полностью их пересматривать
